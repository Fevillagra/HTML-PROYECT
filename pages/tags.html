En línea y En bloque

En línea

Las etiquetas de línea solo ocupan el ancho del elemento como por ejemplo "span"

  • Aparentemente no se puede modificar el ancho de un contenedor "inline"(objeto defectuoso) mediante atributos
  • Si el contenedor de linea tiene adentro un elemento en bloque este no será tan grande como el de bloque a pesar de contenerlo
  • Si adentro tiene un elemento en linea como el contenedor este se adaptara al tamaño del elemento de linea

DIV DE LINEA & ETIQUETA DE LINEA

DIV DE LINEA & ETIQUETA DE BLOQUE


En bloque


Las etiquetas de bloque ocupan todo el ancho de la página

  • Si el contenedor es de LINEA la etiqueta de bloque ocupara toda su linea(o espacio especificado)
  • Si el contenedor es de BLOQUE la etiqueta de bloque ocupara el ancho del contenedor padre

Tipos de etiquetas en HTML5

  • Metadata content: Los elementos que pertenecen a la categoría de contenido de metadatos modifican la presentación o el comportamiento del resto del documento, establecen enlaces a otros documentos o transmiten otra información fuera de banda.
  • Flow content: Flow content is a broad category that encompasses most elements that can go inside the element, including heading elements, sectioning elements, phrasing elements, embedding elements, interactive elements, and form-related elements. It also includes text nodes (but not those that only consist of white space characters).
  • Sectioning content: El contenido de secciones es un subconjunto del contenido de flujo y se puede usar en cualquier lugar donde se espere contenido de flujo. Los elementos que pertenecen al modelo de contenido de sección crean una sección en el esquema actual que define el alcance de los elementos header>, los elementos footer> y el contenido del encabezado.
  • Heading content: El contenido del encabezado es un subconjunto del contenido de flujo, que define el título de una sección, ya sea marcado por un elemento de contenido de sección explícito o definido implícitamente por el propio contenido del encabezado. El contenido del encabezado se puede usar en cualquier lugar donde se espere contenido de flujo.
  • Phrasing content El contenido de fraseo es un subconjunto del contenido de flujo que define el texto y el marcado que contiene, y se puede usar en cualquier lugar donde se espere contenido de flujo. Las rachas de contenido de redacción forman párrafos.
  • Embedded content: El contenido incrustado es un subconjunto del contenido de flujo que importa otro recurso o inserta contenido de otro lenguaje de marcado o espacio de nombres en el documento, y se puede usar en cualquier lugar donde se espere contenido de flujo. Los elementos que pertenecen a esta categoría incluyen: audio, canvas, embed, iframe, img, math, object, picture, svg, video.
  • Interactive content: El contenido interactivo es un subconjunto del contenido de flujo que incluye elementos diseñados específicamente para la interacción del usuario y se puede usar en cualquier lugar donde se espere contenido de flujo. Los elementos que pertenecen a esta categoría incluyen: a, button, details, embed, iframe, keygen>Obsoleto, label, select y textarea. Algunos elementos pertenecen a esta categoría solo bajo condiciones específicas: audio>, if the controls attribute is present img>, if the usemap attribute is present input>, if the type attribute is not in the hidden state menu>, if the type attribute is in the toolbar state object>, if the usemap attribute is present video>, if the controls attribute is present

HTML SEMANTICO

header <header> CONTENIDO </header>

Etiqueta de bloque, se usa para colocar el logo, un formulario de navegacion e incluso tambien el nav a veces

nav <nav> CONTENIDO </nav>

Etiqueta de bloque, contiene enlaces de navegacion

article <article> CONTENIDO </article>

Etiqueta de bloque, contenido principal de la página, suele usarse uno y se divide en secciones

section <section> CONTENIDO </section>

Etiqueta de bloque, subcontenidos del contenedor article

aside <aside> CONTENIDO </aside>

Etiqueta de bloque, Contenido adicional de la página se coloca al lado del article

footer <footer> CONTENIDO </footer>

Etiqueta de bloque, pie de pagina con enlaces de interes, copyright, textos legales, etc.
